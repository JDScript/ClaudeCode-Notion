# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Notion Research Project Management — a Skills-First system for managing
research projects in Notion via Claude Code skills + Notion MCP.

Scoped to the Research Space workspace. No Python code, no build step — all
functionality lives in `.claude/skills/`. Modifying behavior means editing a
skill definition or a template file.

## Notion Workspace

Workspace root page ID is stored in `.claude/notion-config.json` (run `/init` to populate).

Projects follow a standard structure (modeled after Functional Reorientation):
- Project Overview callout + Quick Links callout (two-column layout)
- Inline Tasks DB (schema: Task, Status, Due, Assigned To, Week)
- Inline References DB (schema: Name, Arxiv, Conference, Year, Github, Project Page)
- Some projects have an inline Datasets DB (see `docs/samples/datasets-db-schema.md`)
- Meeting Notes sub-page

## Key Conventions

1. **Content zones**: AI-generated content under `## AI Summary`, user notes under `## My Notes`. Regeneration NEVER touches `## My Notes`.
2. **Template-driven**: All Notion content follows templates in `docs/templates/`. Templates contain a `<!-- TEMPLATE BODY START -->` marker; only content below that marker is written to Notion. Changes take effect immediately on next invocation.
3. **Schema reference**: DB schemas documented in `docs/samples/`.
4. **Functional Reorientation as reference**: New projects mirror its structure in Notion.
5. **Config-driven**: All workspace and project IDs live in `.claude/notion-config.json` (gitignored, per-user). Run `/init` to populate. Skills read this file instead of hardcoding IDs.
6. **Per-project template overrides**: Projects can have a "Templates" sub-page in Notion. Skills check for project-specific templates before falling back to `docs/templates/` defaults.

## File Structure

- `.claude/skills/` — Claude Code skill definitions (SKILL.md per skill, 6 total)
- `.claude/notion-config.json` — Per-user workspace config (gitignored, generated by `/init`)
- `docs/templates/` — Notion page content templates (`paper-summary.md`, `project-page.md`, `meeting-notes.md`, `technical-review-template.md`)
- `docs/samples/` — DB schema snapshots (`tasks-db-schema.md`, `references-db-schema.md`, `datasets-db-schema.md`)
- `docs/architecture/` — System architecture documentation
- `docs/skills/` — Skills catalog and documentation
- `docs/plans/` — Design and implementation plans
